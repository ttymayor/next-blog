## 前言

首先，我自認為我並非絕頂聰明，但很多時候是經驗法則，所以在很多時候我會很常利用過去舊有的經驗去應付現在的問題。還有一部分是<Highlighter>適應力和邏輯能力</Highlighter>，我認為這很重要不過這部分的訓練對我來說仍是經驗和時間練習。

## 淺談背景

我是從國中開始接觸程式，先是 Python 再到後來高中接觸 C/C++，在這過程也接觸了競技程式，簡單來說就是解題。不過我要說的不是解題技巧，我想說的是解題與實作的差別。

解題是在用程式解決問題，所以多半是考演算法居多。

但實作則是有多重考量，包括不侷限於：資料庫、框架、UI/UX、效能、安全、可讀、部屬...。光挑一個都蘊含不少學問。

## 暑假期間發生的一切

很榮幸被教授看好，於是有機會獲得人生第一次的實習經驗。雖說僅僅 13 週（因為時間和身分關係），但我仍舊學習到了許多東西。

在這裡我學習到了許多東西，包括：

- 使用 Laravel
- 使用 Vue.js
- 畫架構圖

### Laravel

Laravel 在我實習之前就有粗略使用過，大概就是除了資料庫外，Route 跟 Controller 的部份。不過更早之前有碰過 Flask 還有一點 Gin，所以對後端還算熟悉，也有接過資料庫。

Laravel 在實習過程大概是：Migration/API → Model → Controller，但新創（你也懂吧）就是塗塗改改，所以最後就是哪裡缺什麼就加什麼。

又因為專案開始變肥，所以最後有接觸到把 Controller 中的 Service 業務邏輯抽出來，需要 Request 的部份就 Request 來做，最後再回傳給 Controller。

### Vue.js

Vue.js 我是真沒碰過，不過 Senior 是拿他原本舊的專案來改，所以大致上我只需要按照原有的邏輯去修改就好。

在碰前端之前，我在高三處理完備審後的閒暇時間，有接觸過網頁三兄弟（HTML、CSS、JavaScript），並且是使用他人的 API 來做一個簡單的網頁，我記得當時寫的是地震報告，至於資料來源...嘻嘻，歡迎來找我聊聊。

## 所以說怎麼短時間學習和使用框架？

我相信現在公司都很希望員工多多使用 AI 工具來輔助開發，不過要想學好一個框架，僅靠 AI 是不夠的，還要有自己的思考和實作，這聽起來很廢話但是事實。

不過有個我在實作時一直保有的概念，就是<Highlighter action="highlight" color="#649300">盡可能的模組化</Highlighter>，這樣在後續維護和擴充時會比較方便。

由於我是從競程開始學程式的，所以習慣了解題方式：當有個需要特殊處理的部分，就先拆解成小問題，再一個一個解決。

所以我更喜歡口語化，我自己的理解是：<Highlighter color="#649300">能不重複寫的東西就不要重複寫</Highlighter>。

因為這樣，我對於資料庫一開始使用時就有想問的：就是我真的要把所有資料存在一張表上？（當然不是）以這個概念去想，就會衍伸出「正規化」的概念。

在學習框架時，我也會盡可能的模組化，這樣在後續維護和擴充時會比較方便。

## AI 協作

我覺得熟悉使用 AI 幫忙固然好，但開發時如果是將程式碼或者專案部分段落貼給 AI 產生，我覺得到最後整個專案的可維護性會下降，所以為了體現自己是工程師專業，我覺得親自審查程式碼以及專案結構十分重要。

使用 Agent 好處是 AI 讀的到前後文，不過某些優化、架構的設計只有你懂，所以去多看看其他人做的專案，了解他們的設計和實作方式，這樣在實作時才不會因為 AI 的建議，而導致自己寫的程式碼難以維護或優化。

## 結語

不敢說自己多會寫程式或使用 AI Agent 協作，但這些都是我認為在學習和實作時很重要的事情。所以總結現在更重視什麼：專案架構、安全性、更深入的技術。
